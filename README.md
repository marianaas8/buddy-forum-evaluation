# 3D Evaluation Scripts

Evaluation pipeline for the three research phases of the dissertation
*"Generative AI for 3D Digital Assistants"* (Mariana Serrão, FEUP, 2026).

Each phase has a dedicated evaluation script. All scripts pull live data from the
Buddy Forum server via `sync_data.py` and optionally log metrics to MLflow.

---

## Repository Structure

```
3d-evaluation/
├── sync_data.py                  # Downloads habit data & user metrics from the server
├── evaluate_3d.py                # Phase 1 — 3D model quality evaluation
├── evaluate_animations.py        # Phase 2 — animation quality evaluation
├── evaluate_layer_selection.py   # Phase 3 — prop layer selection evaluation
├── render_glb.py                 # Blender helper: renders a .glb to a PNG image
├── blender_anim_eval.py          # Blender helper: extracts animation quality metrics
├── blender_prop_eval.py          # Blender helper: computes HOI + kinematic metrics
├── data/
│   ├── eval_list.csv             # Generated by sync_data.py — list of habits + GLB paths
│   ├── metrics.json              # User interaction events (likes, downloads) from server
│   ├── message_metrics.json      # Buddy response ratings from server
│   ├── cortex_logs.json          # Phase 3 only — Cortex prop-selection logs
│   ├── habit_descriptions.json   # Phase 3 only — habit rule descriptions for CLIP
│   ├── habits_metrics.csv        # Output of Phase 1
│   ├── anim_results.csv          # Output of Phase 2
│   └── layer_results.csv         # Output of Phase 3
└── tripo/                        # Standalone latency benchmarks for Tripo AI API
```

## Dependencies

```bash
pip install torch clip-by-openai Pillow trimesh numpy pandas requests mlflow ImageReward nltk
```

- **Blender 5.x** must be installed and the path updated in `BLENDER_EXEC` inside
  `evaluate_animations.py` and `evaluate_layer_selection.py`.
- **gltf-transform** (Node.js) is required by Phase 3 to decompress the Buddy avatar
  GLB for Blender import:
  ```bash
  npm install -g @gltf-transform/cli
  ```
- **MLflow** tracking server is optional. Set `USE_MLFLOW = False` in any script to
  skip it. When enabled, update `MLFLOW_TRACKING_URI` to point to your server.

| `BLENDER_EXEC` | `evaluate_animations.py`, `evaluate_layer_selection.py` | Path to Blender executable |

## Phase 1 — 3D Model Quality (`evaluate_3d.py`)

Evaluates the 3D models generated by Tripo AI from text prompts.

**Metrics computed:**
- Topology: vertex count, face count, watertight status, edge length variance
- Semantic: CLIP score (render vs. text prompt), ImageReward score
- User feedback: likes, dislikes, downloads (from Buddy Forum)

**How to run:**
```bash
python evaluate_3d.py
```

The script calls `sync_data.py` automatically, then for each habit: loads the GLB
with Trimesh, calls Blender headlessly via `render_glb.py` to produce a PNG render,
and scores the render against the text prompt using CLIP and ImageReward.
Results are written to `data/habits_metrics.csv` and logged to MLflow.

---

## Phase 2 — Animation Quality (`evaluate_animations.py`)

Evaluates the skeletal animations assigned to each habit via Tripo AI's rig+retarget
pipeline.

**Metrics computed (skeletal animations only):**
- Geometry: max edge expansion %, mesh clipping ratio, volume variation %, edge stretch variance
- Motion: foot skating ratio, jitter score (L2 acceleration variance)
- Structural: watertight survival (base pose → animated)
- Semantic: CLIP score (mid-frame render vs. animation command), ImageReward
- User feedback: likes, dislikes, downloads

Habits with UI-only animations (float, bounce, spin, pulse) are recorded but skip
Blender evaluation.

**How to run:**
```bash
python evaluate_animations.py
```

Calls Blender headlessly via `blender_anim_eval.py` for each skeletal model.
Results are written to `data/anim_results.csv` and logged to MLflow.

## Phase 3 — Prop Layer Selection (`evaluate_layer_selection.py`)

Evaluates the AI-assisted prop layer selection produced by BrightFactory Cortex.
Only habits created during the Phase 3 evaluation window (2026-05-26 to 2026-05-29)
are processed.

**Metrics computed:**
- Semantic relevance: CLIP Name→Name, CLIP Prompt→Prompt, CLIP Desc→Desc
- Lexical similarity: WordNet Wu-Palmer (WuP) similarity between extracted prop and habit nouns
- Diversity: coverage diversity (mean pairwise cosine distance across selected prop embeddings)
- HOI (Hand-Object Interaction): D2O (distance to object), L2P (limb-to-prop distance),
  Contact Rate, Bone Clearance
- Kinematic: MAD score (mean absolute deviation of prop motion), Jitter score
- Body intersection: intersection rate across 3 animation loops (L1, L2, L3)
- Fallback compliance: filler prop count, theme-aligned filler count
- Latency: Cortex selection latency in seconds
- User feedback: likes, dislikes

**How to run:**
```bash
python evaluate_layer_selection.py
```

Syncs data from the server, fetches Cortex selection logs and habit descriptions,
then calls Blender headlessly via `blender_prop_eval.py` for HOI and kinematic metrics.
Results are written to `data/layer_results.csv` and `data/prop_analysis.json`.

## Helper Scripts

### `sync_data.py`
Downloads habit data from the Buddy Forum server (`/api/habits_sync`), saves GLB
files locally, and fetches user metrics from `/api/metrics` and `/api/metrics/messages`.
Called automatically at the start of each evaluation script.

### `render_glb.py`
Blender Python script (run headlessly) that imports a `.glb`, auto-frames the camera
to the object's bounding box, sets up sun lighting, and renders a 512×512 PNG using
EEVEE.  Called by `evaluate_3d.py`.

### `blender_anim_eval.py`
Blender Python script that loads an animated GLB, samples frames across the full
animation, and computes motion quality metrics (foot skating, mesh clipping, edge
stretch variance, jitter, velocity/acceleration L2 norms).
Called by `evaluate_animations.py`.

### `blender_prop_eval.py`
Blender Python script that imports the Buddy avatar GLB and one or more prop GLBs,
attaches each prop to the avatar's hand bone, and measures HOI metrics (D2O, L2P,
contact rate, bone clearance) and body intersection across three animation loops.
Called by `evaluate_layer_selection.py`.

## Tripo Latency Scripts (`tripo/`)

Standalone scripts for measuring Tripo AI API latency across batches of habits —
used for benchmarking only, not part of the main evaluation pipeline.

| Script | Description |
|---|---|
| `3d_evaluate_latency.py` | Measures text-to-model generation latency per habit |
| `3d_habits_latency.py` | Batch latency test across all habits (3D generation) |
| `anim_evaluate_latency.py` | Measures rig+retarget latency per animated habit |
| `anim_habits_latency.py` | Batch latency test across all habits (animation) |
| `anim_evaluate_all.py` | End-to-end latency: text-to-model + rig + retarget |

---

## Notes

- All scripts require an active network connection to the Buddy Forum server.
  If the server is offline, cached files in `data/` are used where possible.
- CUDA is used automatically if a compatible GPU is available; otherwise CPU is used.
- Phase 3 evaluation with Blender HOI metrics takes several minutes per habit.
- MLflow experiment names: `habit_3D_genai_evaluation` (Phase 1),
  `habit_animation_evaluation` (Phase 2), `habit_layer_selection_evaluation` (Phase 3).
